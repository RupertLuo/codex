Rust 实战 01 · 不要求预习语法

先改一行真正的 Codex，再解释 Rust

这一课只有一个目标：让你亲手完成“运行源码 → 找到一行 → 修改 → 重新编译 → 看见变化 → 恢复现场”。遇到什么语法，我们就只学什么语法。

预计 10–15 分钟 修改 1 行 最终不保留源码改动
只读，不会改文件

确认你正在源码仓库里

先让终端告诉我们当前分支、Rust 和构建工具版本。

cd /Users/luoruipu/projects/other_projects/codex
git status --short --branch
cd codex-rs
rustc --version
cargo --version
just --version

Expect · 预期

分支是 codex/catalyst-transport-v0.142.4，并看到 Rust 1.95.0、Cargo 1.95.0、Just 1.54.0。

Learn · 只学这一点

cd 切换目录；Rust workspace 的构建入口在 codex-rs。

If it fails · 出错

如果提示目录不存在，先运行 pwd，把输出发给我；不要猜路径。

Checkpoint · 检查点

确认 Git 输出里没有以 M、A 或 D 开头的源码文件。

第一次从源码运行

看修改前的帮助文字

just codex --help | rg -n "debug|Debugging"

Expect · 预期

输出包含 debug 和 Debugging tools。第一次构建可能需要几分钟。

Learn · 只学这一点

just codex 会构建并运行当前源码，不是系统里已安装的旧二进制。

If it fails · 出错

如果出现编译错误，只看第一条 error:；把它和上方十行一起发给我。这个发布标签首次构建也可能只改写 Cargo.lock 中的 workspace 版本；运行 git diff -- Cargo.lock 后先停下，我会帮你只恢复这个生成差异。

Checkpoint · 检查点

记住原文 Debugging tools，后面要用同一条命令比较。

仍然只读

找到这行文字来自哪里

rg -n -C 3 "Debugging tools" cli/src/main.rs

Expect · 预期

你会看到 /// Debugging tools.，下面紧跟 Debug(DebugCommand),。

Learn · 只学这一点

rg 是代码搜索工具；-n 显示行号，-C 3 同时显示上下三行。

If it fails · 出错

先运行 pwd。这里应该以 /codex/codex-rs 结尾。

Checkpoint · 检查点

此时我们已经从“屏幕上的文字”追到了“生成它的源码”。

本课唯一一次源码修改

亲手改一行

在编辑器中打开 /Users/luoruipu/projects/other_projects/codex/codex-rs/cli/src/main.rs，只做下面这个替换：

Before

/// Debugging tools.
Debug(DebugCommand),

After

/// Developer diagnostics and debugging tools.
Debug(DebugCommand),

Expect · 预期

git diff -- cli/src/main.rs 只显示一行删除和一行新增。

Learn · 只学这一点

/// 是文档注释，会附着到它下面的 Rust 项目；这里 Clap 用它生成 CLI 帮助。

If it fails · 出错

如果 diff 里还有别的变化，先不要继续，把 diff 发给我。

Checkpoint · 检查点

不要修改 Debug(DebugCommand),；这一轮只改注释。

编译并观察

运行同一条命令

just codex --help | rg -n "debug|Developer diagnostics"

Expect · 预期

帮助文字变成 Developer diagnostics and debugging tools。

Learn · 只学这一点

Rust 先把源码编译成二进制。Just 检测到一行变化后只重建受影响部分。

If it fails · 出错

先运行 git diff -- cli/src/main.rs，确认改动和上一步完全一致。

Checkpoint · 检查点

到这里，你已经完成第一次真实的 Rust 修改和运行。

现在才读两行 Rust

理解刚才碰到的语法

enum Subcommand {
    /// Developer diagnostics and debugging tools.
    Debug(DebugCommand),
}

enum Subcommand 表示“Codex 子命令只能是列出来的这些选择之一”。Debug(...) 是其中一个选择，括号里的 DebugCommand 是这个选择携带的数据。现在不需要理解它内部是什么。

Checkpoint · 一道题

如果把 /// 改成普通的 //，帮助文字还会出现吗？

恢复现场

把练习改动还原

在编辑器里把文档注释改回：

/// Debugging tools.

然后检查：

git diff --check
git status --short

Expect · 预期

两个命令都不显示 Codex 源码改动。

Learn · 只学这一点

git diff 是修改前后的安全网。练习代码不混进正式 transport patch。

If it fails · 出错

不要使用 git reset --hard。把 git diff -- cli/src/main.rs 发给我，我们只恢复这一行。

Checkpoint · 检查点

下一步开始真正的 transport 测试；新代码会按可审查的提交保留。
