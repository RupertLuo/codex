Lesson 01 · Rust for Codex

别把 Rust 当成“难一点的 Python”

Rust 的表面语法并不难。真正的新东西是：每个值由谁负责释放、函数到底拿走了值还是只借来看一下，以及失败是否被类型明确写出来。掌握这三件事，Codex 源码会从“符号墙”变成可以顺着读的程序。

1. 先把表面语法扫清

Rust 是静态类型语言，但局部变量通常能推断类型。变量默认不可变；语句做动作，表达式产生值。

Python

model = "deepseek-chat"
retries = 3
retries += 1

def score(x: int) -> int:
    if x > 0:
        return x * 2
    return 0

Rust

let model = "deepseek-chat";
let mut retries: u32 = 3;
retries += 1;

fn score(x: i32) -> i32 {
    if x > 0 {
        x * 2
    } else {
        0
    }
}

分号规则：块末尾没有分号的表达式就是返回值。写成 x * 2; 后它变成语句，值被丢弃，块的值是 ()，类似 Python 的 None，但它是一个真实的零大小类型。

Rust含义Python 联想
let x = 1;不可变绑定普通赋值，但禁止再次赋值
let mut x = 1;可变绑定普通可变变量
const LIMIT: usize = 8;编译期常量，必须标类型大写模块常量，但 Rust 会强制规则
let x = x.trim();遮蔽：创建同名新绑定名字复用；新值甚至可换类型
fn f(x: i32) -> bool参数类型和返回类型类型标注，但编译器强制
常见标量与容器

i32/u32/i64/u64/usize 是整数，f32/f64 是浮点，另有 bool、char。固定数组是 [T; N]，可增长数组是 Vec<T>，哈希表是 HashMap<K, V>，元组是 (A, B)。安全索引常写 items.get(0)，返回 Option<&T>。

控制流和循环
for item in &items { /* 借用每一项 */ }
for (index, item) in items.iter().enumerate() { /* ... */ }
while ready { /* ... */ }

let answer = loop {
    if done { break 42; }
};

let label = if ok { "ready" } else { "blocked" };

2. 所有权：Rust 真正的主线

Python 对象通常由垃圾回收器追踪。Rust 没有运行时 GC：一个值有一个 owner；owner 离开作用域时，值自动释放。把拥有堆数据的值赋给另一个变量，默认发生 move，不是浅拷贝。

let a = String::from("hello");
let b = a;                 // String 的所有权移动给 b
// println!("{a}");       // 编译错误：a 已失效
println!("{b}");

整数这类固定大小、实现了 Copy 的类型会复制，不会 move。String、Vec<T>、文件句柄等通常不是 Copy。

借用：函数看看，不拿走

拿走所有权

fn send(request: Request) { /* ... */ }

send(request);
// request 此后不能用

只借用

fn validate(request: &Request) { /* ... */ }

validate(&request);
// request 仍然属于调用者

借用的核心规则：同一时间可以有多个 &T 只读借用；或者只有一个 &mut T 可变借用，二者不能重叠。直觉是“很多读者”与“一个写者”二选一。

fn append_system_prompt(prompt: &mut String) {
    prompt.push_str("\nFollow the tool protocol.");
}

let mut prompt = String::from("Research NVDA");
append_system_prompt(&mut prompt);

String 与 &str

String 是拥有、可增长的 UTF-8 字符串；&str 是借来的一段 UTF-8 文本视图。函数只读文本时优先接收 &str。

fn is_deepseek(model: &str) -> bool {
    model.starts_with("deepseek")
}

let owned = String::from("deepseek-chat");
assert!(is_deepseek(&owned));
assert!(is_deepseek("deepseek-reasoner"));

读函数签名的第一问：参数是 T、&T 还是 &mut T？它分别表示“拿走”“只读借用”“独占可变借用”。

生命周期 'a 到底是什么？

引用不能比它指向的数据活得更久。大多数情况编译器会自动推断；当返回引用且来源可能不明确时，生命周期参数描述引用之间的关系。它不会延长任何值的寿命。

fn choose_longer<'a>(a: &'a str, b: &'a str) -> &'a str {
    if a.len() >= b.len() { a } else { b }
}

3. 用类型表达业务状态

struct：数据；impl：行为

#[derive(Debug, Clone)]
struct ModelRequest {
    model: String,
    prompt: String,
    stream: bool,
}

impl ModelRequest {
    fn new(model: impl Into<String>, prompt: impl Into<String>) -> Self {
        Self { model: model.into(), prompt: prompt.into(), stream: true }
    }

    fn is_streaming(&self) -> bool {
        self.stream
    }
}

Self 指当前 impl 的类型。self、&self、&mut self 分别表示方法拿走实例、只读借用实例、可变借用实例。Rust 没有类继承；组合、enum 和 trait 承担建模工作。

enum + match：比 Python 枚举更强

enum StreamEvent {
    TextDelta(String),
    ToolCall { name: String, arguments: String },
    Completed,
    Failed(String),
}

fn render(event: StreamEvent) {
    match event {
        StreamEvent::TextDelta(text) => println!("{text}"),
        StreamEvent::ToolCall { name, arguments } => {
            println!("tool={name} args={arguments}")
        }
        StreamEvent::Completed => println!("done"),
        StreamEvent::Failed(error) => eprintln!("{error}"),
    }
}

match 必须穷尽所有 variant，所以增加协议事件时，遗漏的处理点会变成编译错误。这对 agent event loop 特别有价值。

Option<T>：把“可能没有”写进类型

let temperature: Option<f32> = Some(0.2);

let label = match temperature {
    Some(value) => format!("temperature={value}"),
    None => "temperature=default".to_owned(),
};

if let Some(value) = temperature {
    println!("{value}");
}

Rust 没有可随处使用的 null。Option<T> 迫使调用方处理 Some(T) 或 None。

4. Result：失败也是普通数据

可恢复错误通常不靠异常，而靠 Result<T, E>：成功是 Ok(T)，失败是 Err(E)。

fn load_config(path: &str) -> Result<String, std::io::Error> {
    let text = std::fs::read_to_string(path)?;
    Ok(text)
}

? 的语义是：若为 Ok(value)，取出 value 继续；若为 Err(error)，立刻从当前函数返回一个兼容的 error。它不是“忽略错误”。

写法用途
result?向上传播错误，生产代码最常见
result.unwrap()错误时 panic；测试、原型或逻辑上不可能失败时使用
result.expect("reason")和 unwrap 类似，但 panic 信息更清晰
result.unwrap_or(default)失败时使用默认值
result.map(f)只变换成功值
result.map_err(f)只变换错误值

5. Trait、泛型与 Codex patch seam

trait 类似 Python 的 Protocol/ABC：描述某种行为。struct 通过 impl Trait for Type 实现它。

trait Transport {
    fn name(&self) -> &str;
    fn send(&self, body: &str) -> Result<String, TransportError>;
}

struct DeepSeekTransport { endpoint: String }

impl Transport for DeepSeekTransport {
    fn name(&self) -> &str { "deepseek" }

    fn send(&self, body: &str) -> Result<String, TransportError> {
        todo!()
    }
}

静态分派与动态分派

// 泛型：编译器为具体 T 生成专用代码
fn call<T: Transport>(transport: &T) { /* ... */ }

// impl Trait：上面的简写风格
fn call(transport: &impl Transport) { /* ... */ }

// trait object：运行时可以装入不同 Transport 实现
fn call(transport: &dyn Transport) { /* ... */ }

Box<dyn Transport> 表示堆上拥有一个动态 transport；Arc<dyn Transport + Send + Sync> 表示多个线程/异步任务共享拥有它，而且实现必须能安全跨线程移动与共享。这正是可注入 model transport 常见的形状。

常见智能指针速查
类型用途
Box<T>单一 owner，把值放到堆上；也常用于 trait object
Rc<T>单线程引用计数的共享所有权
Arc<T>原子引用计数，可跨线程共享所有权
Mutex<T>独占锁保护可变数据
RwLock<T>多个读者或一个写者

6. Async：语法短，模型要正确

async fn send_request(client: &Client) -> Result<Response, Error> {
    let response = client.send().await?;
    Ok(response)
}

调用 async fn 不会立即跑完整函数，而是得到一个 Future；.await 允许执行器在等待 I/O 时去推进其他任务。Codex 这类程序通常由 Tokio 提供执行器、异步网络、channel 和 task。

let handle = tokio::spawn(async move {
    run_turn(session).await
});

let result = handle.await??;

async move 会把闭包使用到的值移动进异步任务，确保任务可以比当前函数活得更久。两个 ? 常表示外层 task join 可能失败，内层业务本身也可能返回错误。

Stream 是什么？

Future 最终产生一个值；Stream 随时间产生多个值。模型 SSE 响应天然是 stream：文本 delta、tool call、completed 等事件逐个到达。常见代码会写 while let Some(event) = stream.next().await，逐项处理。

7. 解码一个 Codex 风格类型

Arc<dyn HttpTransport + Send + Sync>
HttpTransport行为合同：能发 HTTP/流式请求
dyn运行时可替换具体实现
Send + Sync允许安全跨任务/线程使用
Arc多个组件共享所有权

合起来就是：“一个可以被多个并发组件共享的、运行时可替换的 HTTP transport”。如果让原生 Codex response 经过自维护协议转换层，这比改写整个 turn loop 更像一个稳定注入点。

模块与可见性

mod transport;              // 声明子模块
pub mod protocol;           // 其他 crate 可见
pub(crate) fn build() {}    // 仅当前 crate 可见

use crate::transport::HttpTransport;
use std::sync::Arc;

Cargo package 可以包含多个 crate；crate 是 Rust 的编译单元。src/lib.rs 通常是 library crate 根，src/main.rs 是 binary crate 根。Codex workspace 有很多 crate，所以读源码时先看当前文件属于哪个 Cargo.toml。

宏与属性

#[derive(Debug, Clone, serde::Serialize)]
struct Request { /* ... */ }

println!("request={request:?}");  // ! 表示宏
let items = vec![1, 2, 3];
let label = format!("model={model}");

宏在编译阶段展开语法。第一次读源码时先把 #[derive(...)] 理解成“请编译器或过程宏替我生成这些能力”即可。
